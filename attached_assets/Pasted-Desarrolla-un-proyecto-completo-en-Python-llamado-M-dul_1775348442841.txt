Desarrolla un proyecto completo en Python llamado "Módulo de Gestión de Inventario", aplicando buenas prácticas de ingeniería de software, arquitectura limpia y diseño profesional.

OBJETIVO:
Crear un sistema que permita gestionar productos, controlar stock, registrar compras y ventas, con una arquitectura escalable, modular y mantenible.

REQUISITOS GENERALES:

* Código altamente modular
* Bajo acoplamiento y alta cohesión
* Aplicar principios SOLID, KISS y DRY
* Uso correcto de encapsulamiento
* Separación clara de responsabilidades

PATRONES DE DISEÑO OBLIGATORIOS:

1. Creacional: Factory Method

   * Usar para crear distintos tipos de productos (ProductoSimple, ProductoPerecedero, etc.)

2. Estructural: Facade

   * Crear una clase que simplifique las operaciones del sistema (ventas, compras, inventario)

3. Comportamental: Observer

   * Implementar un sistema de notificaciones cuando cambie el stock (alertas, logs, etc.)

ARQUITECTURA DEL PROYECTO:
Organizar el código en carpetas:

* models/ → clases de dominio (Producto, Inventario, etc.)
* services/ → lógica de negocio
* controllers/ → flujo de la aplicación
* patterns/ → implementación de Factory, Facade y Observer
* utils/ → utilidades
* main.py → punto de entrada

FUNCIONALIDADES:

* Crear, editar y eliminar productos
* Manejo de stock
* Registrar ventas (disminuye stock)
* Registrar compras (aumenta stock)
* Alertas de stock bajo
* Listar productos

INTERFAZ:

* Implementar una interfaz por consola (CLI)
* Menú interactivo claro y organizado
* Experiencia de usuario simple pero funcional

CÓDIGO:

* Usar programación orientada a objetos
* Usar clases abstractas e interfaces (abc module)
* Código limpio, legible y comentado
* Evitar duplicación de código
* Manejo básico de errores
